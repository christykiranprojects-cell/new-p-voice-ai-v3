+++++++++++++++++++++++++++
Important rule for ML / STT
+++++++++++++++++++++++++++
| Use case                  | Python        |
| ------------------------- | ------------  |
| Whisper + Torch (CPU/GPU) | **3.10.x** ✅ |
| Stable ML research        | **3.11.x** ✅ |
| Experimental / future     | 3.12 (maybe)   |
| 3.13 / 3.14               | ❌ No         |

Why use 3.10.x version only? 
- Python 3.10 is widely used in ML/STT because major libraries (PyTorch, TensorFlow, librosa, numba) 
stabilized and optimized around it. 
- Newer Python releases break C-extensions and CUDA wheels, causing long delays before ML ecosystems 
catch up.
Note: 
- C-extensions: Performance-critical parts of ML libraries written in C/C++ (e.g., PyTorch, NumPy). 
They must be recompiled for every new Python version, causing delays and breakage.
- CUDA wheels: Precompiled GPU-enabled binaries for NVIDIA CUDA. Each Python version needs matching 
CUDA wheels; new Python releases lag months behind.
